## 更完整的解释

2026年，一个反直觉的事实正在成为行业共识：**模型越来越像，但用好模型的方式越来越不一样。**

GPT-5.5 单独 42.7 分，Claude Opus 4.6 单独 43.3 分——差距不到 1 分。但两者协作后 57.4 分，提升不是线性的，是质的飞跃。护城河正在从"谁的模型更强"迁移到"谁的编排更会组合"。

### 为什么编排是护城河

三个结构性原因：

1. **模型在趋同，编排在分化。** 头部模型能力差距已压缩到个位数百分点。正是因为模型各有所长且长板趋同，"怎么组合"才成了决定胜负的变量。模型越像棋子，编排就越像棋手。

2. **编排经验是隐性知识，不可被 API 复制。** 模型可以通过 API 调用复制，但编排经验不能。微软的 Critique 层为什么让 Claude 做审校而不是 GPT 自己做？因为 Claude 在事实核查维度更强，且"生成者"和"审校者"用不同模型能避免同类盲区。这个判断是从大量实测中沉淀的经验。模型是商品化的，编排经验是手艺。

3. **编排系统的切换成本一旦形成，就很难拆。** 企业一旦部署了多模型编排系统，编排层就嵌入了业务流程。模型可以随时换，编排系统换不了——编排策略、上下文工程、工具链适配全部要重来。

### 代价与边界

编排不是免费的：token 消耗是单次对话的 15 倍；编排质量取决于路由策略，路由策略取决于场景理解；跨厂商协作有政治成本。它适合高价值任务——深度研究、大规模批量分析、复杂决策——不适合日常问答。

**机会不在"再做一个 Agent 平台"，在"针对垂直场景做深编排"。** 通用编排比的是工程能力，垂直编排比的是场景理解。后者才是创业者的主场。

## 例子

微软 Copilot Cowork 的核心架构：GPT 写研究报告初稿，Claude 做事实核查和审校，修订后交付。微软投了 OpenAI 130 亿美元，却把最终编辑权交给了 Anthropic 的 Claude。这不是技术选择，是战略选择——编排逻辑优先于模型所有权。微软用行动表态：我的竞争力不在拥有最好的模型，而在拥有最好的模型组合系统。

Manus 被Meta以 20 亿美元收购，买的不是模型权重，是路由策略、上下文工程、工具链整合这些编排能力。这笔交易给编排系统定了一个价：数十亿美元。
