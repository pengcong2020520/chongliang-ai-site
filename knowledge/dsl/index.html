定义：DSL 是面向特定领域设计的表达语言，它用更贴近业务或系统边界的语法，降低复杂任务的表达和执行成本。

适用场景

  • 工作流编排
  • Agent 指令协议
  • 业务规则表达
  • 内容结构生成
  • 评测用例描述

它不是什么

DSL 不一定是完整编程语言，也不一定需要复杂语法；它可以是轻量配置、结构化指令或领域协议。

常见误区

  • 以为 DSL 一定很重
  • 把所有配置都叫 DSL
  • 忽略人的可读性

更完整的解释

自然语言灵活，但边界不稳定；代码稳定，但表达成本高。DSL 位于二者之间，用领域化表达压缩复杂性。

在 AI 系统中，DSL 可以让提示词、工具调用、任务编排和评测描述更稳定。

例子

一个内容生成系统可以用 DSL 描述文章结构、约束和引用要求，让模型输出更接近可发布内容，而不是每次都依赖临时提示词。
