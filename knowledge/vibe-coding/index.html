Vibe Coding 是什么

Vibe Coding 是自然语言驱动的编程范式——你描述感觉，AI 实现代码，编程从一门需要学习的技能变成了一个可以对话的能力。

定义：Vibe Coding 是一种以自然语言意图表达为核心、以三层文档体系（README.md/TODO.md/CLAUDE.md）管理上下文、以 Prompt Caching 优化成本的编程范式，其实质是编程能力从少数人身上被拆解下来平分给每一个人。

适用场景

  • 开发者用自然语言描述需求完成完整项目
  • 非技术人员通过对话构建可用的应用
  • 长周期AI辅助项目的上下文持久化管理
  • 团队协作中AI编程规范的标准化

它不是什么

它不等于"AI 帮你写代码"——Vibe Coding 是 AI 帮你完成整个项目，从需求描述到产品交付；它也不等于不需要工程能力——上下文管理的三层文档体系本身就是工程；更不等于"随便说说就能跑"——不建好文件系统、不维护上下文，AI 会在新对话里失忆。

常见误区

  • 认为 Vibe Coding 不需要工程能力——三层文档体系、Prompt Caching优化、上下文分层都是工程
  • 以为长对话更省事——长对话token消耗是前期的3-5倍
  • 把动态任务状态写进系统级指令导致缓存失效
  • 不建文件系统只靠对话记忆——新对话等于空白记忆

更完整的解释

核心矛盾：长对话贵，短对话忘

用 AI 做项目，最痛的问题不是 AI 不够聪明，而是"失忆"。对话太长，每轮消耗的 token 可能是前期的 3-5 倍；开新对话来"重置"上下文长度，所有项目背景又要重新喂一遍。

唯一的解法是：让项目的关键信息持久化到文件里，AI 每次开新对话自动读取，而不是靠手动喂。

三层文档体系

| 层级 | 文件 | 解决的问题 | 修改频率 | 缓存影响 | |------|------|-----------|---------|---------| | 项目级 | README.md | "这是什么项目？" | 极少修改 | ✅ 高缓存命中 | | 任务级 | TODO.md | "现在在做什么？" | 频繁修改 | ❌ 不需要缓存 | | 对话级 | CLAUDE.md | "这一轮怎么配合？" | 极少修改 | ✅ 高缓存命中 |

CLAUDE.md 是宪法，改一次要全民公投；TODO.md 是日程表，每天都可以划掉几项。别把日程表写进宪法里。

TODO 不能写进 CLAUDE.md，因为 CLAUDE.md 相当于 System Prompt。如果频繁修改，Prompt Caching 会失效，每次调用都要重新处理整个文件，成本翻倍。

上下文持久化的经济学

三层文件系统的精妙之处在于：它把上下文分成了"可缓存的"和"不可缓存的"两部分。

  • 可缓存部分（CLAUDE.md + README.md）：内容稳定，享受 Prompt Caching，缓存命中时处理成本降低 60-80%，开 100 次新对话只付一次完整处理费
  • 不可缓存部分（TODO.md + 当前对话）：内容动态，作为普通上下文读取，但内容量小，成本可控

这不是随意的文件分类，是围绕 Prompt Caching 的经济学做的架构设计。

Vibe Coding 的本质：编程能力平权化

程序员是最先被 AI 淘汰的职业，不是因为 AI 讨厌程序员，而是因为程序员是离 AI 最近的人。从 GPT 发布到 Vibe Coding 成熟，只有四年。

但"替代"不是说程序员消失了。编程能力被从程序员身上拆解下来，平分给了每一个人。就像印刷术让抄写员下岗，但让全人类学会了阅读。Vibe Coding 让程序员下岗，但让全人类学会了编程。

编程从一门需要学 C 语言、数据结构、编译原理的技能，变成了一个可以对话的能力。感觉比精确重要，意图比语法重要，会说话比会写代码重要。

被忽略的翻转：Vibe Coding 是数据采集器

每一次你描述需求、纠正 AI 的输出、说"不对，我要的是这样而不是那样"——这些交互全部是免费的标注数据。你做产品经理时给 AI 描述的产品逻辑，做设计师时描述的审美偏好——这些知识以前锁在你的脑子里，现在通过 Vibe Coding 流进了 AI 的训练管道。

Vibe Coding 不只是一个工具。它是一个数据采集器。采集的，是你对这个岗位的全部理解。

编程只是第一站。所有岗位都会被拆解成一个一个的 Skill——医生看病是"症状输入→知识检索→诊断输出"，律师审合同是"条款输入→规则匹配→风险评估"。这些岗位的逻辑结构与编程没有本质区别。

例子

开发者开新对话时，AI 自动读取三个文件：README.md（知道这是什么项目）、TODO.md（知道当前做到哪了）、CLAUDE.md（知道编码规范）。AI 立刻进入状态，给出实现步骤，确认后动手写代码，完成后更新 TODO.md。一个对话只干一件事——登录一个对话，支付新开一个。发现偷懒立即引用 CLAUDE.md 规则让它重写。这就是 Vibe Coding 在日常工作中的样子——三个文件，三层上下文，四个习惯。
